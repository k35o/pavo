# 出力フォーマットのルール

レビューコメント・スレッド返信の両方に適用される、GitHub Markdown での書き方のルール。

## コード引用

- 識別子・短いシンボル名はインラインコード（バッククォート 1 個）で囲む（例: `useEffect`, `gh pr diff`）
- **バッククォートを含む引用や、複数行・コメント文・文字列リテラルは必ずフェンス付きコードブロック
  （バッククォート 3 個）を使う**。インラインコード内でバッククォートをエスケープしようとしない
  （GitHub Markdown でレンダリングが壊れる）
- ファイル全体の参照は `**path/to/file.ts**` のように bold で示し、続けてフェンス付きコードブロックで
  該当箇所を引用する

## 折りたたみ

- `<details>` を使うときは、開始タグの直後と `</details>` の直前に空行を入れる
  （空行がないと内部の Markdown がレンダリングされない）

## Mermaid

- 図を出すときは `mermaid` 言語指定のフェンス付きコードブロックを使う（GitHub がネイティブレンダリングする）
